0/1 Knapsack Problem (DP)

Either take an item completely or leave it — maximize value within weight capacity using DP.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

Same as fractional knapsack but you cannot take fractions. Each item is either fully included (1) or excluded (0). Greedy fails here — we need DP.

DP State: dp[i][w] = maximum value using first i items with capacity w.

if wt[i] > w: dp[i][w] = dp[i-1][w] (can't include item i)
else: dp[i][w] = max(dp[i-1][w], val[i] + dp[i-1][w-wt[i]]) (exclude or include)

To find which items were selected: Traceback from dp[n][W]: if dp[i][w] != dp[i-1][w], item i was included.

🔑 Key Points
1. Cannot break items — binary decision (0 or 1) for each item.
2. DP table size: (n+1) × (W+1) where n=items, W=capacity.
3. First row and column are all zeros.
4. Time: O(n×W), Space: O(n×W).
5. Answer is at dp[n][W] (bottom-right).
6. Traceback: go from dp[n][W], if value changed → item included, move to dp[i-1][w-wt[i]].
7. Note: O(nW) is pseudo-polynomial — exponential in terms of input bits.
🔍 Worked Example

n=4 items, W=5 capacity. Items: (w=2,v=3), (w=3,v=4), (w=4,v=5), (w=5,v=6)

Item 0:   0 0 0 0 0

Q4 Explain why greedy fails for 0/1 knapsack. W=10, items: (w=5,v=10),(w=5,v=10),(w=6,v=12). Show greedy vs optimal. ▼
GreedyRatios: 10/5=2.0, 10/5=2.0, 12/6=2.0 — all equal. Greedy picks by ratio, say item3(6,12) first, then can't fit both item1,item2 (6+5=11>10). Take item1(5,10): total w=11>10. Take item2 instead: w=6+5=11 still no! Actually item3+item1: 6+5=11>10. Item3 alone=12. Item1+item2=5+5=10 → 20.
Optimal DPdp[3][10]: Take items 1 and 2 (both w=5): v=10+10=20 ✓
Greedy picked item3 first (highest/equal ratio but largest weight) → got stuck with only 12.
Optimal = 20 > 12.
✅ Greedy: 12. DP optimal: 20. Greedy fails for 0/1 knapsack!
