🟠 Greedy Algorithm #4
🔀

Optimal Merge Pattern

Minimize total comparisons when merging sorted files using a greedy min-heap approach.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

You have n sorted files of different sizes. When you merge two files of sizes m and n, the cost is m + n (number of comparisons/operations). You want to merge all files into one with minimum total cost.

This is essentially the same as Huffman coding — use a min-heap and always merge the two smallest files first.

Cost of merging = size1 + size2 | Always merge two smallest → Min Total Cost
💡 Intuition: Small files merged early get "added" into sums multiple times. Merging small files first minimizes how many times large values are included in sums.
🔑 Key Points
1. Use a min-priority queue (min-heap).
2. Each merge costs = sum of two merged files.
3. Repeat n-1 times (each merge reduces file count by 1).
4. Total cost = sum of all intermediate merge costs.
5. This gives the minimum total number of record moves.
6. Same algorithm as Huffman — just different interpretation.
7. Time complexity: O(n log n).
🔍 Worked Example

Files of sizes: {2, 3, 4, 5, 6, 7}

  1. Merge 2+3 = 5, cost=5. Heap: {4,5,5,6,7}
  2. Merge 4+5 = 9, cost=9. Heap: {5,6,7,9}
  3. Merge 5+6 = 11, cost=11. Heap: {7,9,11}
  4. Merge 7+9 = 16, cost=16. Heap: {11,16}
  5. Merge 11+16 = 27, cost=27. Done.
✅ Total Cost = 5+9+11+16+27 = 68
