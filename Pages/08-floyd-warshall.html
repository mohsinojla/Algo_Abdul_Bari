🟢 Graph Algorithm #8
🔁

Floyd-Warshall Algorithm

Find shortest paths between ALL pairs of vertices in a weighted graph.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

Floyd-Warshall computes the shortest path between every pair (i,j) of vertices. It works with negative edges (but not negative cycles).

Key Idea: For each intermediate vertex k, check: "Is the path from i to j shorter if we go through k?"

dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

Repeat this for every possible intermediate vertex k (from 1 to n). This is a DP approach — building from paths with no intermediates up to paths with all possible intermediates.

💡 Remember: The outer loop is over intermediate vertex k, then i, then j. Order matters!
🔑 Key Points
1. Finds all-pairs shortest paths in one run.
2. Time Complexity: O(V³) — three nested loops.
3. Space Complexity: O(V²) for the distance matrix.
4. Works with negative edge weights (unlike Dijkstra).
5. If dist[i][i] < 0 after running → negative cycle exists.
6. Initial matrix: dist[i][j] = weight(i,j) if edge exists, else ∞; dist[i][i]=0.
7. After k iterations, dist[i][j] = shortest path using only vertices {1..k} as intermediates.

Q3 What is the time complexity of Floyd-Warshall and how does it compare to running Dijkstra from every vertex? ▼
Floyd-WarshallThree nested loops over V vertices each → O(V³)
Works with negative edges, simple to implement.
V × DijkstraRun Dijkstra V times: V × O(V²) = O(V³) with adjacency matrix
Or V × O((V+E)logV) with priority queue.
ComparisonSame O(V³) for dense graphs. Dijkstra faster for sparse graphs with priority queue. Floyd-Warshall is simpler and handles negative edges (but no negative cycles).
✅ Both O(V³) for dense graphs. Floyd-Warshall preferred for simplicity and negative edge support.
Q4 After Floyd-Warshall, if dist[2][2] = -3, what does this mean? How do you detect negative cycles? ▼
Meaningdist[2][2] represents the shortest path from vertex 2 back to itself. It should always be 0. If it's negative (-3), it means there's a cycle that includes vertex 2 with negative total weight.
DetectionAfter running Floyd-Warshall, check: if dist[i][i] < 0 for ANY vertex i → negative cycle exists.
In that case, shortest paths are undefined (can always get shorter by traversing the cycle).
✅ dist[i][i] < 0 → negative cycle detected at vertex i!
