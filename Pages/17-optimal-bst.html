Optimal Binary Search Tree

Construct a BST with minimum expected search cost given key probabilities.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

Given n keys k₁ < k₂ < ... < kₙ with probabilities p₁,p₂,...,pₙ (and dummy key probabilities q₀,...,qₙ), build a BST that minimizes expected search cost.

Cost of searching key kᵢ at depth d = (d+1) × pᵢ (root is depth 1).

e[i][j] = expected cost for keys i..j
e[i][j] = min{e[i][r-1] + e[r+1][j] + w(i,j)} for r = i..j
w(i,j) = sum of all p and q from i to j (total probability)
Base: e[i][i-1] = q[i-1]

Intuition: When you add a root r to a subtree containing keys i..j, all depths increase by 1, adding w(i,j) to the cost.

🔑 Key Points
1. e[i][j] = minimum expected search cost for keys i to j.
2. w[i][j] = sum of probabilities of all keys and dummy keys in range i..j.
3. root[i][j] = root of optimal BST for keys i..j (for reconstruction).
4. Time: O(n³), Space: O(n²).
5. More frequent keys should be closer to the root (lower depth).
6. Dummy keys (q values) represent searching for a key that's NOT in the tree.
7. Total probability: Σpᵢ + Σqᵢ = 1.
🔍 Worked Example

n=3 keys: k1, k2, k3 with p=[0.15, 0.10, 0.05] and dummy q=[0.05, 0.10, 0.05, 0.05] (q0..q3)

Wait — for this example, let p₁=0.15, p₂=0.10, p₃=0.05, q₀=0.05, q₁=0.10, q₂=0.05, q₃=0.05

w table: w[i][j] = w[i][j-1] + p[j] + q[j]

w[1][1]=q[0]+p[1]+q[1]=0.05+0.15+0.10=0.30, w[1][2]=0.30+p[2]+q[2]=0.30+0.10+0.05=0.45, w[1][3]=0.45+0.05+0.05=0.55

e table (simplified example):

  • e[1][1]=e[1][0]+e[2][1]+w[1][1]=q[0]+q[1]+w[1][1]=0.05+0.10+0.30=0.45 (root=k1)
  • e[2][2]=q[1]+q[2]+w[2][2]=0.10+0.05+p[2]+q[2]=0.10+0.05+0.10+0.05=0.30
  • e[3][3]=q[2]+q[3]+w[3][3]=0.05+0.05+0.05+0.05=0.20
✅ Build full table continuing for lengths 2,3. The root giving minimum e[1][n] is the optimal root.
❓ Practice Questions
Q1 What is the purpose of dummy keys (q values) in Optimal BST? What do they represent? ▼
MeaningDummy keys represent unsuccessful searches — searching for a value that's NOT in the tree.
qᵢ = probability of searching for a key between kᵢ and kᵢ₊₁.
q₀ = probability of searching for something less than k₁.
qₙ = probability of searching for something greater than kₙ.
ImportanceWithout q values, we only optimize successful searches. Real applications have both successful and unsuccessful searches. Including q makes the optimization realistic and complete.
✅ Dummy keys = failed search probabilities. Ensures the tree is optimized for ALL searches, not just successful ones.
Q2 n=4 keys. Probabilities: p=[0.1, 0.2, 0.4, 0.3]. No dummy keys (q=0). Find optimal BST (which key should be root?). ▼
AnalysisWithout dummy keys, w[i][j]=sum of p[i..j].
e[r][r]=p[r] for single keys.
For e[1][4], we try each r=1,2,3,4 as root:
r=1: e[empty]+e[2][4]+w[1][4]=0+(cost of 2,3,4 as right subtree)+1.0
r=2: e[1][1]+e[3][4]+w[1][4]=0.1+(cost of 3,4)+1.0
r=3: e[1][2]+e[4][4]+1.0=(0.1+0.2+?)+0.3+1.0
r=4: e[1][3]+e[empty]+1.0
IntuitionKey 3 (p=0.4) has highest probability → should be root or near root to minimize cost. Keys searched more often should be at smaller depth.
ResultFull DP gives: Optimal root = k3 (highest probability = lowest expected depth). Expected cost ≈ 1.8
✅ k3 (p=0.4) should be root — most frequent key gets depth 1.
Q3 Why can't we just use a regular BST (e.g., sorted insertion or balanced BST) instead of Optimal BST? ▼
Regular BSTA balanced BST minimizes the maximum depth (O(log n) for all searches). But if some keys are searched 100x more often than others, a balanced tree doesn't exploit this — rare keys get the same depth as frequent ones.
Optimal BSTOptimal BST minimizes the EXPECTED search cost considering probabilities. High-probability keys get smaller depths (closer to root) at the cost of making rare keys deeper. Trade-off is computed exactly by DP.
ExampleIf p1=0.90, p2=0.09, p3=0.01: Optimal BST puts k1 at root (depth 1). Balanced BST puts k2 at root → expected cost is higher since k1 is searched 90% of the time at depth 2 instead of 1.
✅ Regular BST optimizes structure. Optimal BST optimizes for search frequency — minimizes expected cost with given probabilities.
