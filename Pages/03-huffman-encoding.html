Huffman Encoding

Greedy algorithm to create optimal prefix-free codes for data compression.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

Huffman coding assigns shorter binary codes to more frequent characters and longer codes to rare ones. This minimizes the total number of bits used — it's used in ZIP, JPEG, MP3.

Algorithm:

  1. Create a leaf node for each character with its frequency.
  2. Put all nodes in a min-priority queue (min-heap).
  3. Repeat until 1 node remains: extract the two minimum-frequency nodes, create a new internal node with their sum as frequency, make them left and right children, insert back.
  4. The final tree gives the codes: left edge = 0, right edge = 1.
Huffman Code Length = Σ (frequency × code_length) bits
💡 Key Property: Huffman codes are prefix-free — no code is a prefix of another. This makes decoding unambiguous.
🔑 Key Points
1. Min-heap is used — always pick two nodes with minimum frequency.
2. The character with lowest frequency gets the longest code.
3. The resulting tree is called a Huffman Tree (full binary tree).
4. Number of merge operations = n-1 for n characters.
5. Time Complexity: O(n log n) using a min-heap.
6. Total bits = Σ(freq × depth) where depth = length of code for that character.
7. Huffman is optimal — no other prefix-free code gives fewer total bits.
StepComplexity
Build min-heapO(n)
Extract+Insert (n-1 times)O(n log n)
OverallO(n log n)
🔍 Worked Example

Characters: a(5), b(9), c(12), d(13), e(16), f(45)

Min-heap steps:

  1. Extract a(5), b(9) → merge → ab(14). Heap: [c(12), d(13), ab(14), e(16), f(45)]
  2. Extract c(12), d(13) → merge → cd(25). Heap: [ab(14), e(16), cd(25), f(45)]
  3. Extract ab(14), e(16) → merge → abe(30). Heap: [cd(25), abe(30), f(45)]
  4. Extract cd(25), abe(30) → merge → cdabe(55). Heap: [f(45), cdabe(55)]
  5. Extract f(45), cdabe(55) → merge → root(100).
(100) / \ f(45) (55) / \ (25) (30) / \ / \ c d (14) e / \ a b Codes: f=0, c=100, d=101, a=1100, b=1101, e=111
✅ Total bits = 45×1 + 12×3 + 13×3 + 5×4 + 9×4 + 16×3 = 45+36+39+20+36+48 = 224 bits

Q1 Characters with frequencies: A=5, B=1, C=6, D=3, E=4. Build Huffman tree and find codes for each character. ▼
Step 1Min-heap: B(1), D(3), E(4), A(5), C(6)
Step 2Extract B(1)+D(3) → BD(4). Heap:[E(4),BD(4),A(5),C(6)]
Extract E(4)+BD(4) → EBD(8). Heap:[A(5),C(6),EBD(8)]
Extract A(5)+C(6) → AC(11). Heap:[EBD(8),AC(11)]
Extract EBD(8)+AC(11) → root(19).
Step 3Tree: root→left=EBD, right=AC
E=00, B=010, D=011, A=10, C=11
✅ A=10, B=010, C=11, D=011, E=00
Total bits = 5×2+1×3+6×2+3×3+4×2 = 10+3+12+9+8 = 42 bits

Step 1Build Huffman tree (classic example):
Codes: a=0(1 bit), b=101(3), c=100(3), d=111(3), e=1101(4), f=1100(4)

= 45+39+36+48+36+20 = 224 bits per 100 chars

Step 3Fixed 3-bit: 100 chars × 3 = 300 bits per 100
For 100,000 chars: 300,000 bits = 37,500 bytes
Savings = (300,000-224,000)/300,000 ≈ 25.3%
✅ Huffman: 224,000 bits | Fixed: 300,000 bits | Saves ~25%

Q4 How many bits are needed to encode "aababcabc" using Huffman coding? Frequencies: a=5, b=3, c=1. ▼
Step 1Count: a=5, b=3, c=1. Min-heap: c(1),b(3),a(5)
Step 2Merge c(1)+b(3)→cb(4). Heap:[a(5),cb(4)]
Merge a(5)+cb(4)→root(9)
Step 3Codes: a=0(1 bit), b=11(2 bits), c=10(2 bits)
Encode "aababcabc": a+a+b+a+b+c+a+b+c
= 0+0+11+0+11+10+0+11+10 = 0,0,11,0,11,10,0,11,10
Bits = 5×1 + 3×2 + 1×2 = 5+6+2 = 13 bits
✅ 13 bits needed (vs 9×2=18 bits with fixed 2-bit encoding)
