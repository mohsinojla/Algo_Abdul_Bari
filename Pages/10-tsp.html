🚗

Travelling Salesman Problem (TSP)

Find the shortest Hamiltonian cycle that visits all cities exactly once and returns to start.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

A salesman must visit n cities, starting and ending at the same city, visiting each city exactly once. Goal: minimize total travel distance.

Brute Force: Try all (n-1)! permutations → O(n!) — impossible for large n.

DP Approach (Held-Karp): Use bitmask DP to represent which cities have been visited.

dp[S][i] = minimum cost to visit all cities in set S, ending at city i, starting from city 0

Recurrence: dp[S][i] = min over all j in S\{i}: dp[S\{i}][j] + cost[j][i]

⚠️ TSP is NP-Hard — no polynomial algorithm is known. DP gives O(2ⁿ × n²) which is exponential but much better than O(n!). For exams, usually n ≤ 4 or 5.
🔑 Key Points
1. TSP is NP-Hard — believed no polynomial solution exists.
2. Exact DP solution: O(2ⁿ × n²) time, O(2ⁿ × n) space.
3. A Hamiltonian cycle = visits every vertex exactly once and returns to start.
4. For small n (≤4), can enumerate all permutations directly.
5. The DP state uses a bitmask to represent the set of visited cities.
6. Base: dp[{0}][0] = 0 (at start city, cost 0).
7. Final answer: min over all i: dp[{all cities}][i] + cost[i][0].

Q2 Using DP Held-Karp notation, define g(S,i) for TSP. For 3 cities {1,2,3} starting at 1, show how to compute g({2,3},2) and g({2,3},3). ▼
Definitiong(S,i) = minimum cost path starting at vertex 1, visiting all vertices in set S exactly once, ending at vertex i.
Base casesg({2},2) = cost[1][2]
g({3},3) = cost[1][3]
(Only city 1 was the start, now reached city 2 or 3)
Recurrenceg({2,3},2) = g({3},3) + cost[3][2] (come from 3 to 2, set S\{2}={3})
g({2,3},3) = g({2},2) + cost[2][3] (come from 2 to 3)
FinalOptimal = min(g({2,3},2)+cost[2][1], g({2,3},3)+cost[3][1])
✅ DP builds up: base cases → intermediate sets → full set → add return cost
Q3 Why is TSP considered NP-Hard? How many tours are there for n=10 cities? ▼
NP-HardTSP is NP-Hard because: (1) No polynomial-time algorithm is known. (2) All known exact algorithms are exponential. (3) It's as hard as the hardest problems in NP — every NP problem can be reduced to it.
CountFor n cities, fixing start: (n-1)!/2 distinct tours (dividing by 2 for direction).
n=10: 9!/2 = 362880/2 = 181,440 tours.
n=20: 19!/2 ≈ 60 quadrillion tours!
✅ n=10 → 181,440 tours. Grows as O(n!) — computationally intractable.
