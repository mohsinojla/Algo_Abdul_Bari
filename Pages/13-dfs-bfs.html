DFS and BFS Traversals

Two fundamental ways to explore all vertices in a graph or tree.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview
🟠 DFS — Depth First Search

Go as deep as possible before backtracking. Uses a stack (or recursion).

DFS(v):
  mark v visited
  for each neighbor u of v:
    if u not visited:
      DFS(u)  // go deeper
🔵 BFS — Breadth First Search

Explore all neighbors at current level before going deeper. Uses a queue.

BFS(start):
  enqueue start, mark visited
  while queue not empty:
    v = dequeue
    for each neighbor u:
      if not visited:
        enqueue u, mark visited
🔑 Key Points
1. DFS uses a Stack (LIFO); BFS uses a Queue (FIFO).
2. Both visit every vertex exactly once: Time O(V+E), Space O(V).
3. BFS gives shortest path (fewest edges) in unweighted graphs.
4. DFS is used for: topological sort, cycle detection, connected components, maze solving.
5. BFS is used for: shortest path, level-order traversal, social network distance.
6. DFS can be recursive (implicit stack) or iterative (explicit stack).
7. For disconnected graphs: run DFS/BFS from every unvisited vertex.
🔍 Worked Example

Graph (adjacency list, start=1): 1:[2,3], 2:[4,5], 3:[6], 4:[], 5:[], 6:[]

DFS from 1:
  1. Visit 1, push neighbors [2,3]
  2. Visit 2 (stack: [3]), push 4,5
  3. Visit 4 (stack: [3,5])
  4. Visit 5 (stack: [3])
  5. Visit 3 (stack: []), push 6
  6. Visit 6
Order: 1,2,4,5,3,6
BFS from 1:
  1. Enqueue 1. Queue:[1]
  2. Dequeue 1, enqueue 2,3. Queue:[2,3]
  3. Dequeue 2, enqueue 4,5. Queue:[3,4,5]
  4. Dequeue 3, enqueue 6. Queue:[4,5,6]
  5. Dequeue 4. Queue:[5,6]
  6. Dequeue 5,6.
Order: 1,2,3,4,5,6

Q3 What data structure does DFS use? What does BFS use? How does this affect the traversal order? ▼
DFSStack (LIFO) — Last In, First Out.
The most recently discovered neighbor is explored next → goes DEEP before wide.
In recursion: the call stack acts as the implicit stack.
BFSQueue (FIFO) — First In, First Out.
Neighbors discovered first are explored first → explores level by level, going WIDE before deep.
This guarantees shortest path (fewest hops) in unweighted graphs.
✅ DFS=Stack (deep first) | BFS=Queue (level by level) | BFS finds shortest path in unweighted graphs
