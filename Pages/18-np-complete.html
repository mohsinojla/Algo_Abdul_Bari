NP-Hard & NP-Complete Problems

Understanding computational complexity classes and the Clique Decision Problem.

▶ Watch Abdul Bari on YouTube
📖 Complexity Classes Overview
NP-Hard
At least as hard as hardest NP problems. May not be in NP.
NP (Non-deterministic Polynomial)
Solutions verifiable in polynomial time.
P (Polynomial)
Solvable in polynomial time. P ⊆ NP ⊆ NP-Hard

NP-Complete = NP ∩ NP-Hard (hardest problems in NP)

P (Polynomial)
  • Solvable in O(nᵏ) time
  • Examples: Sorting, BFS, Dijkstra, LCS
  • Tractable problems
NP (Nondeterministic Poly)
  • Solution verifiable in poly time
  • May or may not be solvable in poly time
  • P ⊆ NP (all P problems are in NP)
NP-Complete
  • In NP + NP-Hard
  • Every NP problem reduces to it
  • Examples: SAT, Clique, Vertex Cover, TSP decision
  • Solve one → solve all NP
NP-Hard
  • At least as hard as NP-Complete
  • May not be in NP (not necessarily decidable)
  • Examples: TSP optimization, Halting Problem
  • No known poly-time solution
🔑 Key Points
1. P vs NP question: Does P = NP? The greatest unsolved problem in CS. ($1M prize!)
2. Reduction: Problem A reduces to B (A ≤p B) means: if we can solve B, we can solve A.
3. To prove NP-Complete: (1) Show it's IN NP (verifiable in poly time). (2) Show an NP-Complete problem reduces to it.
4. First NP-Complete problem: SAT (Boolean Satisfiability) — Cook's Theorem (1971).
5. NP-Complete implies: No poly-time algorithm known, but solution can be verified quickly.
6. For NP problems in practice: use approximation algorithms, heuristics, or exact algorithms for small n.
7. Clique problem is NP-Complete (proved by reducing from SAT or 3-SAT).
🕸️ Clique Decision Problem

A clique in a graph is a subset of vertices where every pair is connected by an edge (complete subgraph).

Clique Decision Problem (CDP): "Does graph G contain a clique of size k?"

Graph G: 1 --- 2 |\ /| | X | |/ \| 4 --- 3 (all 6 edges present: 1-2, 1-3, 1-4, 2-3, 2-4, 3-4) This is a 4-CLIQUE (K₄): all 4 vertices are pairwise connected. Smaller example: 1 --- 2 --- 3 | 4 Clique of size 2: {1,2}, {2,3}, {2,4} Clique of size 3: {1,2,3}? Check: 1-2 ✓, 2-3 ✓, 1-3? ✗ → NOT a clique

Why NP-Complete?

  • In NP: Given a clique candidate, verify in O(k²) — just check all edges.
  • NP-Hard: 3-SAT ≤p Clique (polynomial reduction exists — proved by Cook/Karp).
❓ Practice Questions
Q1 Define P, NP, NP-Hard, and NP-Complete. Give one example of each. ▼
PP: Problems solvable in polynomial time O(nᵏ).
Example: Sorting (O(n log n)), Shortest Path (Dijkstra O(V²))
NPNP: Problems where a given solution can be VERIFIED in polynomial time.
Example: Hamiltonian Cycle — can verify a proposed cycle in O(V)
NP-HardNP-Hard: At least as hard as any NP problem. Every NP problem reduces to it.
Example: TSP optimization problem, Halting Problem
NP-CompleteNP-Complete: Both in NP AND NP-Hard. Hardest problems in NP.
Example: Clique decision, SAT, Vertex Cover, 3-Coloring
✅ P⊆NP. NP-Complete = NP ∩ NP-Hard. If any NP-Complete problem is solvable in poly time → P=NP.
Q2 Find all cliques in this graph: vertices {1,2,3,4,5}, edges: {1-2,1-3,2-3,3-4,4-5,3-5}. What is the maximum clique size? ▼
CheckCliques of size 2 (every edge): {1,2},{1,3},{2,3},{3,4},{4,5},{3,5} — all are 2-cliques.
Size 3Check {1,2,3}: 1-2✓, 1-3✓, 2-3✓ → YES! 3-clique!
Check {3,4,5}: 3-4✓, 4-5✓, 3-5✓ → YES! 3-clique!
Check {1,2,4}: 1-2✓, 1-4? No edge → not a clique
Check {1,3,5}: 1-3✓, 3-5✓, 1-5? No → not a clique
Size 4Check {1,2,3,4}: need 1-4,2-4 → no edges → not a clique
Check {1,2,3,5}: need 1-5,2-5 → no → not a clique
No 4-clique exists.
✅ Maximum clique size = 3. Two max cliques: {1,2,3} and {3,4,5}
Q3 How do you prove a problem is NP-Complete? Explain the two conditions with the Clique problem as example. ▼
Condition 1Show the problem is in NP: Given a certificate (candidate solution), verify it in polynomial time.
For Clique: Given a set S of vertices as candidate clique, check if |S|=k and all pairs in S are connected. This takes O(k²) = polynomial → Clique is in NP ✓
Condition 2Show it's NP-Hard: Reduce a known NP-Complete problem to it in polynomial time.
For Clique: 3-SAT ≤p Clique (Karp reduction, 1972). For each 3-SAT formula with k clauses, construct a graph G such that G has a k-clique iff the formula is satisfiable. This reduction is polynomial.
ConclusionClique ∈ NP (condition 1) + Clique ∈ NP-Hard (condition 2) → Clique is NP-Complete!
✅ To prove NP-Complete: (1) In NP: verifiable in poly time. (2) NP-Hard: known NP-C problem reduces to it.
Q4 What is the difference between NP-Hard and NP-Complete? Is TSP NP-Hard or NP-Complete? Explain. ▼
DifferenceNP-Hard: At least as hard as any NP problem. NOT required to be in NP.
NP-Complete: Must be BOTH in NP (verifiable in poly time) AND NP-Hard.
So: NP-Complete ⊂ NP-Hard. Every NP-Complete is NP-Hard, but not vice versa.
TSPTSP has two versions:
1. TSP Decision: "Is there a tour with cost ≤ k?" → NP-Complete (in NP: verify tour, AND 3-SAT reduces to it)
2. TSP Optimization: "Find minimum cost tour" → NP-Hard (not obviously in NP since verifying optimality is hard)
✅ TSP Decision = NP-Complete. TSP Optimization = NP-Hard. All NP-Complete problems are NP-Hard but not the reverse.
