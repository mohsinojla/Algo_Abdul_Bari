Bellman-Ford Algorithm

Single-source shortest paths that handles negative edge weights and detects negative cycles.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

Bellman-Ford finds the shortest path from one source to all other vertices. Unlike Dijkstra, it handles negative edge weights and can detect negative cycles.

Algorithm:

  1. Initialize: dist[source]=0, dist[all others]=∞.
  2. Repeat V-1 times: Relax ALL edges — for each edge (u,v,w): if dist[u]+w < dist[v], update dist[v].
  3. Negative cycle check: Do one more relaxation pass. If any distance still decreases → negative cycle exists.
Relax: if dist[u] + w(u,v) < dist[v] → dist[v] = dist[u] + w(u,v) | Repeat V-1 times
⚠️ Why V-1 iterations? A shortest path with no negative cycles has at most V-1 edges. After k iterations, all shortest paths with ≤k edges are found.
🔑 Key Points
1. Relax ALL E edges in each of V-1 iterations.
2. Time: O(V × E) — slower than Dijkstra but handles negatives.
3. If after V-1 iterations, doing one more relaxation still updates → negative cycle detected.
4. Works on both directed and undirected graphs.
5. Space: O(V) for distance array.
