Dijkstra's Algorithm

Find shortest paths from a single source to all vertices in a weighted graph.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

Dijkstra's algorithm finds the shortest path from a source vertex to all other vertices in a graph with non-negative edge weights.

Algorithm:

  1. Initialize: dist[source]=0, dist[all others]=∞.
  2. Use a visited set. Repeat until all vertices visited:
  3. Pick the unvisited vertex with minimum distance (greedy choice).
  4. Relax all edges: if dist[u] + weight(u,v) < dist[v], update dist[v].
  5. Mark u as visited.
Relaxation: dist[v] = min(dist[v], dist[u] + w(u,v))
⚠️ Important: Dijkstra FAILS with NEGATIVE edge weights. Use Bellman-Ford for negative edges.
🔑 Key Points
1. Works only with non-negative edge weights.
2. Greedy — always processes the minimum distance unvisited vertex next.
3. Once a vertex is visited/finalized, its distance never changes.
4. Time: O(V²) with adjacency matrix, O((V+E) log V) with priority queue.
5. Produces a shortest path tree from source.
6. Can find the actual path by maintaining a parent/predecessor array.
ImplementationTimeSpace
Adjacency MatrixO(V²)O(V²)
Priority QueueO((V+E) log V)O(V+E)

Q3 Why does Dijkstra fail with negative edges? Give a small counter-example. ▼
ReasonDijkstra assumes: once a vertex is finalized (visited), its shortest distance is permanently set. With negative edges, a later path could DECREASE an already-finalized distance — violating the greedy assumption.
ExampleGraph: A→B(1), A→C(3), C→B(-3)
Dijkstra: Visit A→B(1) finalized. Visit C(3). C→B would give 3+(-3)=0 < 1 but B is already finalized!
Dijkstra gives dist[B]=1 but correct answer is 0.
✅ Use Bellman-Ford for graphs with negative edges!
