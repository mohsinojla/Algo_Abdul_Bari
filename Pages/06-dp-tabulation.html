Dynamic Programming — Tabulation

Bottom-up DP: build solutions to sub-problems iteratively in a table.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

Dynamic Programming solves problems by breaking them into overlapping sub-problems and storing results to avoid recomputation.

Tabulation (Bottom-Up): Start from the smallest sub-problem and build up to the original problem. Fill a table iteratively — no recursion needed.

🔵 Tabulation (Bottom-Up)
  • Fill table from base cases up
  • Iterative (loops)
  • No recursion stack overflow
  • Generally faster in practice
  • Must compute ALL sub-problems
🟠 Memoization (Top-Down)
  • Recursive + cache results
  • Only computes needed sub-problems
  • More natural/intuitive
  • Risk of stack overflow for deep recursion
  • Uses a memo/cache dictionary
Tabulation: dp[i] = f(dp[i-1], dp[i-2], ...) — fill left to right
🔑 Key Points
1. Identify overlapping sub-problems — if recursive solution recomputes same values, DP applies.
2. Define the DP state — what does dp[i] or dp[i][j] represent?
3. Write the recurrence relation — how does dp[i] depend on smaller sub-problems?
4. Establish base cases — fill smallest values first.
5. Fill order matters — each cell must be computed before cells that depend on it.
6. Usually achieves polynomial time vs exponential for naive recursion.
🔍 Worked Example — Fibonacci

fib(n) = fib(n-1) + fib(n-2), fib(0)=0, fib(1)=1

Tabulation for n=7:

n01234567
dp[n]011235813
dp[0]=0, dp[1]=1
for i in 2..n:
    dp[i] = dp[i-1] + dp[i-2]
return dp[n]   // Time: O(n), Space: O(n)
✅ fib(7) = 13 | Time: O(n) vs Naive O(2ⁿ)
❓ Practice Questions
Q1 Using DP tabulation, compute the number of ways to climb n=5 stairs, if you can take 1 or 2 steps at a time. ▼
Statedp[i] = number of ways to reach stair i
Recurrence: dp[i] = dp[i-1] + dp[i-2]
Base: dp[0]=1 (stand at ground), dp[1]=1
Filldp[2]=dp[1]+dp[0]=1+1=2
dp[3]=dp[2]+dp[1]=2+1=3
dp[4]=dp[3]+dp[2]=3+2=5
dp[5]=dp[4]+dp[3]=5+3=8
✅ 8 ways to climb 5 stairs
Q2 Coin change problem: coins={1,3,5}, amount=7. Find minimum number of coins using tabulation. ▼
Statedp[i] = min coins to make amount i
dp[0]=0, dp[i]=∞ initially
dp[i] = min(dp[i-c]+1) for each coin c ≤ i
Filldp[0]=0
dp[1]=dp[0]+1=1 (coin 1)
dp[2]=dp[1]+1=2 (coin 1)
dp[3]=min(dp[2]+1,dp[0]+1)=1 (coin 3!)
dp[4]=dp[3]+1=2
dp[5]=min(dp[4]+1,dp[2]+1,dp[0]+1)=1 (coin 5!)
dp[6]=dp[5]+1=2
dp[7]=min(dp[6]+1,dp[4]+1,dp[2]+1)=min(3,3,3)=3? Wait: dp[7]=dp[6]+1=3 or dp[4]+1=3 or min(dp[2]+1)=3
Answerdp[7]=3 (e.g., 5+1+1 or 3+3+1)
✅ Minimum 3 coins (e.g., 5+1+1)
Q3 What are the 4 steps to solve any DP problem? Apply them to find the n-th Tribonacci number where T(n)=T(n-1)+T(n-2)+T(n-3). ▼
Step 1Identify sub-problems: T(n) depends on T(n-1), T(n-2), T(n-3) — overlapping!
Step 2Define state: dp[i] = i-th Tribonacci number
Step 3Recurrence: dp[i] = dp[i-1]+dp[i-2]+dp[i-3]
Base cases: dp[0]=0, dp[1]=1, dp[2]=1
Step 4Fill table for n=7:
dp=[0,1,1,2,4,7,13,24]
dp[3]=0+1+1=2, dp[4]=1+1+2=4, dp[5]=1+2+4=7, dp[6]=2+4+7=13, dp[7]=4+7+13=24
✅ T(7) = 24 | Time: O(n), Space: O(n) or O(1)
Q4 Longest Increasing Subsequence (LIS) using tabulation. Array: [3, 1, 8, 2, 5]. Find LIS length. ▼
Statedp[i] = length of LIS ending at index i
Base: dp[i]=1 for all i (each element alone)
Filldp[0]=1 (3)
dp[1]=1 (1, nothing smaller before)
dp[2]=2 (3,8 or 1,8 — 1 element before 8 is smaller)
dp[3]=2 (1,2)
dp[4]=3 (1,2,5)
Answermax(dp) = max(1,1,2,2,3) = 3 → subsequence [1,2,5]
✅ LIS = 3, subsequence: [1, 2, 5]
