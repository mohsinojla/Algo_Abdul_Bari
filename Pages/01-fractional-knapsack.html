🟠 Greedy Algorithm #1
🎒

Fractional Knapsack Problem

A greedy approach to maximize the value you can carry by taking fractions of items.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

You have a knapsack with a weight capacity W and a set of n items, each with a weight and a value. You want to maximize the total value in the knapsack.

In the Fractional Knapsack, you can take fractions of items (unlike 0/1 Knapsack). Think of it like filling a bag with liquids — you can pour half a container.

The greedy strategy: always pick the item with the highest value-per-unit-weight (profit/weight ratio) first. Keep taking until the knapsack is full.

Ratio = Value / Weight  →  Sort by Ratio (descending) → Fill Greedily
💡 Why Greedy works here: Each unit of weight has an "exchange rate" — profit per kg. Taking the highest exchange rate first always gives the optimal solution. This is provably optimal (exchange argument).
🔑 Key Points
1. Sort items by value/weight ratio in descending order.
2. Take items one by one — if item fits completely, take it. Otherwise take the fraction that fills remaining capacity.
3. Stop when knapsack is full (remaining capacity = 0).
4. Greedy gives the optimal solution for fractional version.
5. Time Complexity: O(n log n) for sorting + O(n) for filling = O(n log n).
6. Space Complexity: O(1) extra (in-place).
7. Greedy does NOT work for 0/1 Knapsack — use DP for that.

🔍 Worked Example

Given: Capacity W = 50 kg, 5 items:

Sorted by ratio: Item1(6) → Item2(5) → Item3(4) → Item4=Item5(3)

Filling:

  • Take Item1 fully: weight used = 10, value = 60, remaining = 40
  • Take Item2 fully: weight used = 20, value = 100, remaining = 20
  • Take Item3 partially: 20/30 fraction → value = 120×(20/30) = 80, remaining = 0
✅ Maximum Value = 60 + 100 + 80 = $240
❓ Practice Questions
Q1 Knapsack capacity = 15 kg. Items: A(2kg,$10), B(3kg,$5), C(5kg,$15), D(7kg,$7), E(1kg,$6). Find the maximum profit using fractional knapsack. ▼
Step 1Calculate ratio for each item:
A: 10/2 = 5.0 | B: 5/3 = 1.67 | C: 15/5 = 3.0 | D: 7/7 = 1.0 | E: 6/1 = 6.0
Step 2Sort by ratio descending:
E(6.0) → A(5.0) → C(3.0) → B(1.67) → D(1.0)
Step 3Fill greedily (capacity=15):
• Take E: 1kg → value=6, remaining=14
• Take A: 2kg → value=10, remaining=12
• Take C: 5kg → value=15, remaining=7
• Take B: 3kg → value=5, remaining=4
• Take D partially: 4/7 fraction → value = 7×(4/7) = 4, remaining=0
✅ Max Profit = 6 + 10 + 15 + 5 + 4 = $40
Q2 W = 60. Items: (10,$120), (20,$60), (30,$100). Apply fractional knapsack and find max value. ▼
Step 1Calculate ratios:
Item1: 120/10 = 12 | Item2: 60/20 = 3 | Item3: 100/30 = 3.33
Step 2Sorted: Item1(12) → Item3(3.33) → Item2(3)
Step 3Fill (capacity=60):
• Item1: 10kg → value=120, remaining=50
• Item3: 30kg → value=100, remaining=20
• Item2: 20kg → value=60, remaining=0
✅ Max Value = 120 + 100 + 60 = $280
Q3 W = 50. Items: (10,$60), (40,$40), (20,$100), (30,$120). Find maximum value. (Classic Abdul Bari example) ▼
Step 1Ratios: (10kg,60$)=6.0 | (40kg,40$)=1.0 | (20kg,100$)=5.0 | (30kg,120$)=4.0
Step 2Sorted: 6.0 → 5.0 → 4.0 → 1.0
Step 3Fill (capacity=50):
• Item(10kg): value=60, remaining=40
• Item(20kg): value=100, remaining=20
• Item(30kg) partially: 20/30 → value=120×(2/3)=80, remaining=0
✅ Max Value = 60 + 100 + 80 = $240
Q4 W = 20 kg. Items: (18kg,$25), (15kg,$24), (10kg,$15). Find max profit. Note: ratios are very close — careful sorting! ▼
Step 1Ratios: 25/18=1.39 | 24/15=1.60 | 15/10=1.50
Step 2Sorted: 1.60 → 1.50 → 1.39
(15kg item) → (10kg item) → (18kg item)
Step 3Fill (capacity=20):
• (15kg,$24): value=24, remaining=5
• (10kg,$15) partially: 5/10 → value=15×0.5=7.5, remaining=0
✅ Max Profit = 24 + 7.5 = $31.5
Q5 Explain why greedy fails for 0/1 Knapsack. Give a counter-example with W=10, items: (6,$6), (5,$5), (5,$5). ▼
Step 1Ratios: (6kg,$6)=1.0 | (5kg,$5)=1.0 | (5kg,$5)=1.0 — all equal.
Greedy picks first item (6kg,$6), leaving 4kg. Can't fit any other item.
Greedy result = $6
Step 2Optimal 0/1: Skip 6kg item, take both 5kg items: 10kg total.
Optimal result = $10
Step 3In 0/1 Knapsack, you cannot break items. A locally good choice (high ratio) may block better combinations. Greedy ≠ optimal for 0/1.
✅ Greedy gives $6 but optimal is $10 — hence DP is needed for 0/1 Knapsack.
