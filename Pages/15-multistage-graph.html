Multistage Graph

Find the shortest path from source to target in a directed graph with stages using DP.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

A multistage graph is a directed, weighted graph where vertices are partitioned into k stages. Edges go only from stage i to stage i+1. Source is in stage 1, target in stage k.

Goal: Find the minimum cost path from source (s) to target (t).

Two approaches:

  • Forward approach: Compute cost[i] = min cost from vertex i to target. Work backwards from stage k-1 to stage 1. cost[v] = min(w(v,u) + cost[u]) for all u reachable from v.
  • Backward approach: Compute bcost[i] = min cost from source to vertex i. Work forward.
Forward: cost[v] = min{ c(v,u) + cost[u] } for all edges (v,u)
Answer: cost[s]
🔑 Key Points
1. Vertices grouped into k stages; edges only go forward (stage i → i+1).
2. Forward approach: Start from destination, compute min cost working backwards.
3. Backward approach: Start from source, compute min cost working forwards.
4. Time Complexity: O(V+E) — each vertex and edge processed once.
5. Track p[i] (next vertex in optimal path) for path reconstruction.
6. Base case (forward): cost[target] = 0.
7. DP is applicable because shortest path satisfies Principle of Optimality.
🔍 Worked Example (Classic 4-stage graph)
Stage 1: {1} Stage 2: {2,3,4} Stage 3: {5,6,7} Stage 4: {8} Edges: 1→2(9), 1→3(7), 1→4(3) 2→5(2), 2→6(3) 3→5(6), 3→6(5) 4→5(8), 4→7(2) 5→8(6) 6→8(5) 7→8(5)

Forward DP (backward from target):

  • cost[8] = 0 (target)
  • Stage 3: cost[5] = 5+6=? → cost[5]=min(c(5,8)+cost[8])=6, cost[6]=5, cost[7]=5
  • Stage 2: cost[2]=min(c(2,5)+cost[5], c(2,6)+cost[6])=min(2+6,3+5)=min(8,8)=8 → p[2]=5 or 6
    cost[3]=min(6+6,5+5)=min(12,10)=10 → p[3]=6
    cost[4]=min(8+6,2+5)=min(14,7)=7 → p[4]=7
  • Stage 1: cost[1]=min(9+8, 7+10, 3+7)=min(17,17,10)=10 → p[1]=4
✅ Min cost = 10 | Path: 1 → 4 → 7 → 8 (cost: 3+2+5=10)

Q2 Explain forward vs backward approach for multistage graph. When is each preferred? ▼
Forward DPcost[v] = minimum cost from v to target.
Compute from last stage backwards to source.
cost[v] = min{c(v,u)+cost[u]} for all neighbors u
Answer: cost[source]
Backward DPbcost[v] = minimum cost from source to v.
Compute from source forward to target.
bcost[v] = min{bcost[u]+c(u,v)} for all u in previous stage
Answer: bcost[target]
PreferenceBoth give same answer. Forward is more common in textbooks. Backward is like Dijkstra's approach. Use whichever is more natural for the given problem.
✅ Forward: work from target backwards. Backward: work from source forwards. Both O(V+E).

Q4 What makes a graph a "multistage graph"? Can BFS/Dijkstra solve it? Why use DP? ▼
DefinitionA multistage graph has:
1. Vertices partitioned into k disjoint stages
2. Edges only go from stage i to stage i+1 (no backward/skip edges)
3. Single source (stage 1) and single sink (stage k)
BFS vs DPDijkstra/BFS can solve it but DP is more efficient for this specific structure.
DP exploits the stage structure: O(V+E) vs Dijkstra's O((V+E)logV).
The stage property guarantees no cycles → topological order for DP.
✅ DP is optimal for multistage graphs: exploits stage structure, O(V+E), no cycles.
