Principle of Optimality

Bellman's foundational principle that justifies dynamic programming.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview
Bellman's Principle of Optimality (1957):
"An optimal policy has the property that whatever the initial state and initial decision are, the remaining decisions must constitute an optimal policy with regard to the state resulting from the first decision."

In simpler words: Every sub-solution of an optimal solution is also optimal.

If you have the shortest path from A to C via B, then the sub-path A→B must also be the shortest path from A to B, and B→C must be the shortest from B to C.

This property is called "optimal substructure" and is the key requirement for DP to work.

🔑 Key Points
1. Stated by Richard Bellman in 1957 as the foundation of Dynamic Programming.
2. A problem has optimal substructure if optimal solutions contain optimal sub-solutions.
3. This is what separates DP problems from problems where greedy or brute force is required.
4. Proof by contradiction: If a sub-solution were not optimal, we could replace it with a better one → contradicts that the overall solution is optimal.
5. Problems satisfying this: Shortest path, LCS, 0/1 Knapsack, Matrix Chain Multiplication.
6. Problems NOT satisfying this: Longest simple path in graphs (NP-hard), TSP with specific constraints.
7. Combined with overlapping sub-problems, optimal substructure → DP is applicable.
🔍 Examples of Optimal Substructure
✅ Shortest Path (satisfies principle)

Shortest path A→D via B,C: A→B→C→D.
Sub-paths A→B, A→C, B→C, B→D, C→D are all individually shortest.
If B→C were not shortest, replace it → contradicts A→D being shortest.

✅ 0/1 Knapsack (satisfies principle)

Optimal selection of items for capacity W includes an optimal selection for capacity W-wt[i] using the remaining items. Each sub-decision is optimal.

❌ Longest Simple Path (does NOT satisfy)

Longest simple path from q to t (no repeated vertices): q→r→t.
But individual sub-paths might not be longest simple paths due to vertex restrictions.
Hence no polynomial DP solution exists for this problem.

❓ Practice Questions
Q1 State the Principle of Optimality. Why is it important for Dynamic Programming? ▼
Statement"In an optimal sequence of decisions, each sub-sequence must also be optimal."
Equivalently: An optimal solution to a problem contains optimal solutions to its sub-problems.
ImportanceWithout optimal substructure, we cannot combine optimal sub-solutions to get an optimal overall solution. DP works by: (1) solving sub-problems once, (2) storing results, (3) combining them. This only gives correct answers if sub-solutions compose optimally.
✅ Optimal substructure → DP gives optimal overall solution by combining stored sub-solutions.
Q2 Prove by contradiction that shortest path satisfies the Principle of Optimality. ▼
SetupLet P* = optimal (shortest) path from u to v passing through intermediate vertex w.
So P* = P(u,w) + P(w,v).
ContradictionAssume P(u,w) is NOT the shortest path from u to w.
Then ∃ shorter path P'(u,w) with cost less than P(u,w).
Then P' + P(w,v) would be shorter than P* — contradicting that P* is optimal!
ConclusionTherefore P(u,w) MUST be shortest path from u to w. Same argument for P(w,v). QED.
✅ Proof complete: Shortest path sub-paths are all shortest → Principle of Optimality holds.
Q3 Give 3 examples of problems that satisfy the Principle of Optimality and 2 that do NOT. ▼
Satisfy ✅1. Shortest Path (Dijkstra, Bellman-Ford, Floyd-Warshall)
2. 0/1 Knapsack — optimal subset for W contains optimal for W-w[i]
3. LCS — LCS of X[1..m],Y[1..n] contains LCS of sub-strings
4. Matrix Chain Multiplication — optimal split contains optimal splits
5. Fibonacci — F(n) depends on optimal F(n-1), F(n-2)
Do NOT ❌1. Longest Simple Path — LPath(u,v) via w doesn't mean LPath(u,w) and LPath(w,v) are independently longest (can't reuse vertices)
2. 0/1 Knapsack with items that must be paired — custom constraints can break substructure
✅ Shortest path, 0/1 Knapsack, LCS, MCM satisfy. Longest simple path does NOT.
