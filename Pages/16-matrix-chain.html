Matrix Chain Multiplication

Find the optimal parenthesization to minimize scalar multiplications.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

Given a chain of n matrices A₁A₂...Aₙ, we want to find the most efficient way to parenthesize them to minimize total scalar multiplications.

Key fact: To multiply a (p×q) matrix by a (q×r) matrix takes p×q×r multiplications.

The order of multiplication affects cost dramatically but NOT the result.

m[i][j] = min cost to multiply matrices i through j
m[i][j] = min{ m[i][k] + m[k+1][j] + p[i-1]*p[k]*p[j] } for k = i..j-1
Base: m[i][i] = 0
💡 p[] array: For n matrices, we need n+1 dimensions. Matrix Aᵢ has dimensions p[i-1] × p[i].
🔑 Key Points
1. m[i][j] = minimum multiplications to compute the product of matrices i through j.
2. Fill the table by chain length: length 1, 2, 3, ..., n.
3. s[i][j] = optimal split point k (used to reconstruct the parenthesization).
4. Time: O(n³), Space: O(n²).
5. The answer is in m[1][n].
6. Dimensions array has n+1 elements: p[0],p[1],...,p[n].
7. Reconstruct using s table: s[1][n] gives first split, then recurse on sub-problems.
🔍 Worked Example

4 matrices: A₁(10×30), A₂(30×5), A₃(5×60), A₄(60×10)

p = [10, 30, 5, 60, 10] (dimensions array)

Length 1: m[1][1]=m[2][2]=m[3][3]=m[4][4]=0

Length 2:

  • m[1][2]: k=1: p[0]*p[1]*p[2]=10*30*5=1500
  • m[2][3]: k=2: p[1]*p[2]*p[3]=30*5*60=9000
  • m[3][4]: k=3: p[2]*p[3]*p[4]=5*60*10=3000

Length 3:

  • m[1][3]: k=1: m[1][1]+m[2][3]+10*30*60=0+9000+18000=27000; k=2: m[1][2]+m[3][3]+10*5*60=1500+0+3000=4500 → m[1][3]=4500
  • m[2][4]: k=2: m[2][2]+m[3][4]+30*5*10=0+3000+1500=4500; k=3: m[2][3]+m[4][4]+30*60*10=9000+0+18000=27000 → m[2][4]=4500

Length 4:

  • m[1][4]: k=1: m[1][1]+m[2][4]+10*30*10=0+4500+3000=7500; k=2: m[1][2]+m[3][4]+10*5*10=1500+3000+500=5000; k=3: m[1][3]+m[4][4]+10*60*10=4500+0+6000=10500 → m[1][4]=5000
✅ Minimum multiplications = 5000 | Optimal: ((A₁A₂)(A₃A₄)) at k=2

Length 1m[1][1]=m[2][2]=m[3][3]=0
Length 2m[1][2]: k=1: p[0]*p[1]*p[2]=10*100*5=5000
m[2][3]: k=2: p[1]*p[2]*p[3]=100*5*50=25000

k=1: m[1][1]+m[2][3]+p[0]*p[1]*p[3]=0+25000+10*100*50=25000+50000=75000

✅ Min = 7500 multiplications | Parenthesization: (AB)C = (A₁A₂)A₃

L=4m[1][4]:
k=1: 0+104+5*4*7=104+140=244
k=2: 120+84+5*6*7=204+210=414
k=3: 88+0+5*2*7=88+70=158

✅ Min multiplications = 158 | Optimal: ((A₁A₂A₃)A₄) → split at k=3
Q3 Why does the order of matrix multiplication matter? Show with 2 examples: A(2×3)×B(3×4)×C(4×1). ▼
Option 1(AB)C: First AB: 2×3×4=24 mults → result is 2×4. Then (AB)C: 2×4×1=8 mults. Total=32
Option 2A(BC): First BC: 3×4×1=12 mults → result is 3×1. Then A(BC): 2×3×1=6 mults. Total=18
ConclusionBoth give the SAME result matrix (2×1) but different costs: 32 vs 18. The right parenthesization saves ~44% operations! For larger chains the difference is enormous.
✅ Optimal: A(BC) = 18 mults. Order matters hugely for efficiency!
