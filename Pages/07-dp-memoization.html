Dynamic Programming — Memoization

Top-down DP: recursive solution with a cache to avoid redundant computation.

▶ Watch Abdul Bari on YouTube
📖 Concept Overview

Memoization is the "top-down" approach to DP. You write a normal recursive function, but cache (remember) the result of each sub-problem the first time it's computed. If the same sub-problem is encountered again, return the cached value instead of recomputing.

3-Step Process:

  1. Write the naive recursive solution.
  2. Add a memo dictionary/array.
  3. Before computing: check if memo has the answer. After computing: store in memo.
memo[n] = result → if already computed, return memo[n] immediately
🔑 Key Points
1. Top-down approach — start with original problem, recurse to smaller ones.
2. Cache stores: memo[input] = result.
3. Only computes sub-problems that are actually needed (lazy evaluation).
4. Each unique sub-problem is solved exactly once.
5. Without memo: fib(n) = O(2ⁿ). With memo: fib(n) = O(n).
6. Memo can be a dictionary (Python), array, or hash map.
7. Risk: stack overflow for very deep recursion (use tabulation then).
🔍 Worked Example — Fibonacci with Memo

Without memo: fib(5) calls fib(3) twice, fib(2) three times...

fib(5) → fib(4) + fib(3) fib(4) → fib(3) + fib(2) ← fib(3) computed AGAIN fib(3) → fib(2) + fib(1) ← fib(2) computed again... Total calls: 2ⁿ (exponential!)

With memo:

fib(5) → fib(4) + fib(3)[cached=2] fib(4) → fib(3) + fib(2)[cached=1] fib(3) → fib(2) + fib(1) ← computed ONCE, stored fib(2) → fib(1)+fib(0) = 1 ← computed ONCE Total unique calls: n (linear!)
memo = {}
def fib(n):
    if n <= 1: return n
    if n in memo: return memo[n]   # Cache hit!
    memo[n] = fib(n-1) + fib(n-2) # Compute + store
    return memo[n]
✅ fib(5) = 5 | Without memo: 15 calls | With memo: 5 calls

fib(4) calls fib(3)→[cache hit!] and fib(2).
fib(3) calls fib(2)→[cache hit!] and fib(1).
fib(2)=fib(1)+fib(0)=1.
CountUnique computations: fib(0),fib(1),fib(2),fib(3),fib(4),fib(5) = 6 unique sub-problems.
Cache hits: fib(4) in fib(5), fib(3) in fib(4), fib(2) in fib(3) = many reuses.
✅ 6 unique sub-problems. Without memo: 25 calls. With memo: 6 calls!

Q3 What is the time and space complexity of memoized fibonacci? Compare with naive recursion and tabulation. ▼
NaiveTime: O(2ⁿ) — exponential (recomputes same values)
Space: O(n) recursion stack
MemoizationTime: O(n) — each sub-problem solved once
Space: O(n) for memo + O(n) recursion stack = O(n)
TabulationTime: O(n) — same as memoization
Space: O(n) for table, O(1) if optimized (only keep last 2 values)
No recursion stack risk!
✅ Both memo and tabulation: O(n) time. Tabulation is usually preferred for space optimization.
